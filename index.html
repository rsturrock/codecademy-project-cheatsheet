Semantic HTML

Semantic - "relating to meaning"

By using semantic html, we select elements based on their meaning, not how they are presented.

Why use Semantic HTML?

  • accessbility

  • seo

  • easy to understand

Examples of Semantic HTML and Their Uses

Tag Name Formatted Tag Use
nav <nav> is used to define a block of navigation links such as menus and tables of content
main <main> used to encapsulate the dominant content
footer <footer> content at the bottom of the subject information
section <section> defines elements in a document; chapters, headings
article <article> holds content that make sense on its own

CSS Selectors

CSS Selectors - define the elements to which a set of CSS rules apply.

By using CSS Selectors, we are able to select and apply styles to all of our elements without needing to manually style each individual element.

Why use CSS Selectors

  • time efficient

  • best practices

  • easy to understand

Examples of CSS Selectors and Their Uses

Selector Name Formatted Selector Use
type type a selector is used to target the specific HTML elements to be styled by the declaration. type selector matches the type of element in the HTML document
universal * selects all elements of any type. uses the * character
class . to select an HTML element by its class using CSS, a period (.) must be prepended to the class's name
id # selects an HTML element by id attribute selected by prepending with #
psuedo-class : can be attached to any selector. it is always written as a colon : followed by a name
